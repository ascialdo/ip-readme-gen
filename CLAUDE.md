# CLAUDE.md — ip-readme-gen

Developer guide for Claude Code. Read this before making any changes.

---

## What this tool does

Injects two sections into the `README.md` of a VHDL IP core repository:

- **`## Module Interface`** — generics table + ports table, extracted from the VHDL top entity file. Register-mapped ports are excluded (they are exposed via AXI, not directly). An `AXI4-Lite Slave` row is always added at the end of the ports table.
- **`## Register Mapping Information`** — content embedded from `<entity>_axi/<entity>_regmap.md`, generated by `axi-wrapper-gen`.

If a section already exists in the README, its content is replaced in place. If it does not exist, it is appended. The tool is idempotent — running it multiple times produces the same result.

## Repository structure

```
cli.py                        # Entry point — loads config, parses VHD, drives injection
pyproject.toml                # Package definition, installs ip-readme-gen command
ir/
  model.py                    # Copied from axi4lite-wrapper-generator (Port, Generic, Direction, etc.)
parser/
  vhdl_parser.py              # Copied from axi4lite-wrapper-generator (regex-based VHDL parser)
injector/
  readme.py                   # Section injection logic + markdown content builders
tests/
  test_readme_gen.py          # Full test suite (33 tests)
```

> **Note**: `ir/model.py` and `parser/vhdl_parser.py` are intentional copies from
> `axi4lite-wrapper-generator`. This keeps the tool standalone with zero external dependencies.
> If the upstream parser is updated, sync manually.

## Key API

```python
# injector/readme.py

inject_section(text: str, heading: str, content: str) -> str
# Replaces content under `## heading` until the next `##`, or appends if not found.

build_module_interface(ports, generics, mapped_ports=None, register_width=32) -> str
# Builds the ## Module Interface markdown content.
# mapped_ports: set of port names to exclude (they are AXI register fields).
# Always appends an AXI4-Lite Slave row at the end of the ports table.

build_register_mapping(regmap_path) -> str
# Reads _regmap.md, strips the # title, downgrades ## to ### (critical — see below).

build_register_mapping_warning(entity_name: str) -> str
# Returns a markdown warning block when _regmap.md is not found.
```

---

## Critical design decisions

### 1. `##` → `###` downgrade in embedded regmap content

`_regmap.md` uses `## REG_NAME` headings for individual registers. When embedded inside
`## Register Mapping Information`, these `##` headings would be mistaken for section
boundaries by `inject_section`, causing content to be partially replaced and duplicated
on repeated runs.

**Fix**: `build_register_mapping()` downgrades all `## ` to `### ` before returning the content.
This is also semantically correct: registers are sub-sections of the mapping section.

**Never remove this step.**

### 2. Mapped ports are excluded from the ports table

Ports listed as `port` in any `registers.*.fields[*]` entry of `axi_config.json` are
excluded from the Module Interface ports table. They are accessible via the AXI4-Lite
slave interface and listing them separately would be misleading.

The `AXI4-Lite Slave` row is always added regardless of whether any ports are mapped.

### 3. Zero external dependencies

The tool has no runtime dependencies beyond Python stdlib. `jinja2` is NOT used.
`ir/model.py` and `parser/vhdl_parser.py` are copied, not imported from the other package,
to avoid `parser` and `ir` top-level package conflicts in site-packages.

## Config file

The tool reads `axi_config.json` from the current working directory — the same file used
by `axi-wrapper-gen`. No separate config needed.

```json
{
  "top_entity": "rtl/my_entity.vhd",
  "register_width": 32,
  "registers": {
    "REG_CTRL": {
      "offset": "0x00",
      "fields": [
        { "port": "enable", "bits": [0, 0], "access": "RW" }
      ]
    }
  }
}
```

## Intended CI/CD order in an IP repository

```yaml
- axi-wrapper-gen    # generates VHDL + <entity>_axi/<entity>_regmap.md
- ip-readme-gen      # injects Module Interface and Register Mapping sections into README.md
```

## Error behaviour

| Condition | Behaviour |
|-----------|-----------|
| `axi_config.json` missing | Hard error, exit 1 |
| `top_entity` key missing | Hard error, exit 1 |
| VHD file not found | Hard error, exit 1 |
| `README.md` missing | Hard error, exit 1 |
| `_regmap.md` not found | Warning printed, warning block injected into README |
