# ip-readme-gen

A CLI tool that injects a **Module Interface** section and a **Register Mapping Information** section into the `README.md` of a VHDL IP core repository.

Designed to run as a CI/CD stage, after [`axi-wrapper-gen`](https://github.com/ascialdo/axi4lite-wrapper-generator) has already generated the register map documentation.

## Output

Updates two sections in `README.md`:

- **`## Module Interface`** — generics and ports extracted from the VHDL top entity file
- **`## Register Mapping Information`** — embedded from `<entity>_axi/<entity>_regmap.md`

If a section already exists, its content is replaced. If it does not exist, it is appended.

If `_regmap.md` is not found (i.e. `axi-wrapper-gen` has not run yet), a warning is embedded in the Register Mapping section instead of erroring.

---

## Usage

#### 1. Install

```bash
pip install git+https://github.com/ascialdo/ip-readme-gen
```

#### 2. Requirements in your IP repository

- `axi_config.json` at the root (same file used by `axi-wrapper-gen`)
- `README.md` at the root
- Optionally: `<entity>_axi/<entity>_regmap.md` generated by `axi-wrapper-gen`

#### 3. Run

```bash
ip-readme-gen
```

---

## axi_config.json format

Same file used by `axi-wrapper-gen`:

```json
{
  "top_entity": "rtl/my_entity.vhd",
  "register_width": 32,
  "registers": { ... }
}
```
